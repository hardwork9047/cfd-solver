# サロゲート支援トポロジー最適化による耐ファウリング膜形状の探索 — 研究計画

LBM-DEM 連成シミュレーションを評価器として、膜ファウリングが**起きにくい膜形状**(および対極の**起きやすい膜形状**)をトポロジー最適化で探索する研究の計画書。

関連: [`RESEARCH_ROADMAP.md`](RESEARCH_ROADMAP.md)(メカニズム解明ロードマップ。本計画の Phase 0 は同 Phase 1 の解析指標と共通基盤)

---

## 1. 問題設定 — 多目的パレート探索として定式化する

**「ファウリングしにくい膜」の自明解は「穴の大きい筒」である。**
ファウリング最小化を単目的にすると、最適化は細孔を開く方向に退化する。したがって本研究は次の 3 目的の**パレートフロント探索**として定式化する:

| 目的 | 定義 | データ源 |
|---|---|---|
| f1: 初期透過性 | クリーン状態の流束(ウォームアップ後の `u_max` / passed_rate 初期値) | `time_series.csv` |
| f2: 粒子阻止率 | 膜としての分離性能(捕捉粒子数 / 発生粒子数) | `summary.json` |
| f3: 透過性維持率 | T ステップ後の相対透水性(= ファウリング耐性) | `time_series.csv` の passed_rate 比 |

**科学的な問い:** 「同じ阻止率・同じ初期流束で、ファウリングだけが遅い形状は存在するか?」

対極(ファウリングが速い形状)もパレート面の探索で自然に得られる。両極端の比較が、メカニズム解明(千鳥配置 vs 整列、のど部の数と幅、淀み点と付着位置の関係)に直結する。

---

## 2. なぜサロゲートモデルが核心か

| 制約 | 帰結 |
|---|---|
| 3D 1 ケース = 数時間〜200 時間 | 最適化に必要な数百〜数千評価は直接実行不可能 |
| DEM の付着イベントは離散的・確率的 | 随伴法(adjoint)による形状勾配が取れない → 微分不要の手法が必須 |
| 粒子シーディングの乱数で結果がばらつく | ノイズを明示的に扱うサロゲート(noisy GP)が必要 |

→ **サロゲートベースの多目的ベイズ最適化**が正攻法。

---

## 3. 設計空間(段階導入)

形状はソルバー内で `{x, y, radius}` の円柱リスト → ボクセル solid mask に変換される
(`src/particulate_flow/lbm3d.py` の `_build_cylinder_solid`)。

| Stage | 設計空間 | 次元 | 必要な実装 |
|---|---|---|---|
| A | 円柱 N 本の配置・半径(x, y, r) | 10〜30 | なし(既存 config そのまま) |
| B | 構造化パラメータ(千鳥度、列数、半径分布、のど幅プロファイル) | 5〜15 | ジェネレータのみ |
| C | 自由形状ボクセルマスク(真のトポロジー最適化) | 10³〜10⁴ | `_build_cylinder_solid` の任意マスク対応 |

制約条件: 円柱非重複、最小のど幅(粒子が物理的に通れる下限)、空隙率レンジ、流入/流出部のクリアランス。

## 4. サロゲートモデルの構成(3 層)

### 層 1: ガウス過程 + 多目的ベイズ最適化(まずこれ)

- 設計変数 = Stage A/B のパラメトリック空間(≤30 次元)
- **BoTorch の qNEHVI**(ノイズ対応・多目的獲得関数)を使用
- 評価高価・ノイズあり・低次元 → GP-BO の教科書的適用領域

### 層 2: 幾何記述子ベースの回帰(設計則の抽出)

形状から物理的に意味のある特徴量を計算し、GBM 等でファウリング指標を回帰:

- 空隙率、最小のど幅/粒径比、迂曲度、収縮率、固体表面積、淀み領域面積

精度よりも**「何が膜をファウリングさせるか」の解釈可能な設計則**を得るのが目的。
これ自体が研究成果の柱になる。

### 層 3: CNN/U-Net サロゲート(自由形状フェーズ用)

- solid mask 画像 → ファウリング指標(or 付着粒子の空間分布場)を予測
- 微分可能サロゲートとして勾配ベースの自由形状最適化への橋渡し
- 学習データが数百件貯まってから着手

### マルチフィデリティ戦略(計算量の桁を落とす鍵)

| 低忠実度 | 高忠実度 | 成立条件(Phase 0 で検証) |
|---|---|---|
| 2D ソルバー | 3D ソルバー | 2D/3D の指標相関が十分高いこと |
| 短時間ラン(例 25k steps) | 完走(125k steps) | 初期ファウリング速度が長時間挙動を予測すること |

## 5. 実験基盤 — スイープインフラをデータ生成エンジンとして再利用

```
設計ベクトル
  → ジェネレータ(新規スクリプト 1 本)
      configs/lbm_dem/geometries/ フラグメント + ケース JSON を自動生成(extends 機構を利用)
  → マニフェスト自動生成(configs/lbm_dem/sweeps/manifests/)
  → ./bin/sweep run で複数マシン分散実行(git 同期)
  → sweep_results/ に軽量結果が集約
  → time_series.csv / summary.json から f1, f2, f3 を算出
  → サロゲート(GP)更新 → BO が次の設計を提案 → ループ
```

進捗監視は Web UI(`./bin/sweep-web`)がそのまま使える。

## 6. フェーズ計画

### Phase 0 — 成立性検証(数週間)

1. **ノイズ測定:** 同一形状・乱数シード違いの 3〜5 反復で分散を測る
   → 必要反復数と GP のノイズ項が決まる
2. **時間方向フィデリティ検証:** 短時間ラン(25k)と完走(125k)の指標相関を確認
3. **次元方向フィデリティ検証:** 2D vs 3D の指標相関を数ケースで確認
4. **ジェネレータ実装:** 設計ベクトル → ケース JSON 自動生成スクリプト

> 実行中の `fouling_3d_20260609`(12 ケース)は、ノイズ・指標感度の事前情報としてそのまま活きる。

### Phase 1 — 2D スクリーニング(設計則)

- ラテン超方格サンプリングで 50〜100 形状を 2D 評価
- 層 2 の記述子回帰で支配因子を特定
- 仮説検証: 「のど幅/粒径比」「千鳥度」が支配的か?

### Phase 2 — 多目的 BO ループ(本番)

- GP-BO(qNEHVI)で 2D パレートフロントを構築
- BO 提案 → ケース生成 → sweep 実行 → ハーベスト → GP 更新 のループを半自動化
- パレート上の代表形状のみを 3D で検証

### Phase 3 — 自由形状への拡張

- `_build_cylinder_solid` を任意ボクセルマスク対応に拡張
- CNN サロゲート + 進化的探索(CMA-ES)or 微分可能サロゲート勾配
- 円柱の枠を超えた形状(翼型ピラー、非対称のど、深さ方向の傾斜構造)を探索

## 7. リスクと対策

| リスク | 対策 |
|---|---|
| 自明解(大穴)への収束 | 阻止率の下限制約 + パレート定式化(§1) |
| ノイズ過大で形状差が埋もれる | Phase 0 で分散測定。反復平均 or 共通乱数(同一シード列で形状だけ変える) |
| 2D と 3D の乖離 | Phase 0-3 で相関確認。低ければ 2D はスクリーニング専用と割り切り 3D 比重を上げる |
| ケーキ層透過抵抗の未モデル化 | 相対比較に徹する。必要なら ROADMAP Phase 3 の Brinkman 拡張と合流 |

---

## 8. 期待される成果

1. **設計則:** 幾何記述子とファウリング指標の定量関係(解釈可能モデル)
2. **パレートマップ:** 透過性 × 阻止率 × ファウリング耐性のトレードオフ地図
3. **最適形状:** 耐ファウリング膜 / 易ファウリング膜の具体的形状とそのメカニズム説明
4. **方法論:** LBM-DEM + 多目的 BO + マルチフィデリティの探索フレームワーク自体
