# Sweep Web UI 使い方ガイド

複数マシンに分散したパラメータスイープを、ブラウザから監視・実行・停止し、結果の保存先まで確認できる軽量 Web アプリの使い方をまとめます。

CLI(`./bin/sweep`)と同じ仕組みを薄くラップしたものなので、**実行ロジック・git 同期・結果ハーベストはすべて CLI 側**が担います。Web UI はその制御と可視化のフロントです。

---

## 1. セットアップ(初回のみ)

```bash
# Web 依存(FastAPI / uvicorn）を追加インストール。コア sim には影響しないオプショングループ
poetry install --with web
```

> 通常の `poetry install` ではこのグループはスキップされます。Web UI を使うマシンだけで `--with web` を実行してください。

---

## 2. 起動

```bash
./bin/sweep-web
# -> Sweep Web UI -> http://127.0.0.1:8765  (Ctrl-C to stop)
```

ブラウザで **http://127.0.0.1:8765** を開きます。

| 環境変数 | 既定値 | 用途 |
|---|---|---|
| `SWEEP_WEB_HOST` | `127.0.0.1` | 待ち受けホスト。LAN 内の別端末から見たい場合は `0.0.0.0` |
| `SWEEP_WEB_PORT` | `8765` | ポート番号 |

```bash
# 例: LAN 内の他端末からアクセスできるようにする
SWEEP_WEB_HOST=0.0.0.0 SWEEP_WEB_PORT=9000 ./bin/sweep-web
```

## 3. 画面の見方

```
┌────────────────────────────────────────────────────────────┐
│ Sweep Web UI   [スイープ選択 ▼]   ローカルで実行/停止…       │  ← ヘッダ
├────────────────────────────────────────────────────────────┤
│ Cases                                                        │
│ [▶ 選択を実行] [■ 停止]  □force ☑no-push ☑no-install        │  ← 操作バー
│ ┌──┬─────┬──────────┬─────────┬────────┬──────┬──────────┐  │
│ │☐ │Alias│Case      │Status   │Machine │When  │Progress  │  │  ← 状況テーブル
│ │☐ │b0   │..baseline│pending  │ -      │ -    │ –        │  │
│ │☐ │a1   │..attr0015│in-prog. │pc-beta │09:12 │▓▓░ 60%   │  │
│ │☐ │a2   │..attr0010│done     │pc-alpha│23:41 │ –        │  │
│ └──┴─────┴──────────┴─────────┴────────┴──────┴──────────┘  │
├──────────────────────────┬─────────────────────────────────┤
│ Log [run_id]             │ Results [case]                  │
│ [2/2] メインシミュレー…  │ /path/to/sweep_results/...       │  ← ログ & 結果
│ DONE smoke3d ...         │ final_particle_count: 324  …     │
│                          │ [particle_count vs step グラフ]  │
└──────────────────────────┴─────────────────────────────────┘
```

### ステータスバッジの意味

| バッジ | 状態 | 判定元 |
|---|---|---|
| 🟢 `done` | 完了・結果ハーベスト済み | `sweep_results/<sweep>/<case>/run_status.json` |
| 🔵 `in-progress` | どこかのマシンで実行中 | `STARTED.json` |
| 🔴 `failed` | 失敗 | `FAILED.json` |
| ⚪ `pending` | 未実行 | 何もなし |

- **Status / Machine / When は全マシン分**が見えます(git 同期された `sweep_results/` が情報源)。
- **Progress バーは「このマシンで実行中」のケースだけ**表示されます(VTK スナップショット数から算出、`snapshot_every` ステップごとに更新)。

## 4. 基本的な使い方

### スイープを実行する

1. ヘッダのプルダウンでスイープを選ぶ(例 `fouling_3d_20260609`)。
2. 実行したいケースの**チェックボックス**を入れる(このマシンの担当分を選ぶ)。
3. オプションを必要に応じて切り替える(下表)。
4. **[▶ 選択を実行]** を押す。

選んだケースは `./bin/sweep run` が**逐次**実行し、各ケース完了ごとに軽量結果をハーベスト → git commit(/push)します。

| オプション | 意味 | 既定 |
|---|---|---|
| `--force` | 完了済みケースも再実行 | OFF |
| `--no-push` | commit はするが push しない | **ON** |
| `--no-install` | 起動時の `poetry install` をスキップ(速い) | **ON** |

> 他マシンと結果を共有したい場合は `--no-push` を**外して**ください。ON のままだとローカル commit に留まります。

### 進捗を見る

- 実行中ケースの行に進捗バー(`60% (60/100)` のように現在/予想スナップショット数)が出ます。
- 下部 **Log** パネルに実行ログがリアルタイム(2.5秒間隔)で流れます。

### 停止する

- **[■ 停止]** を押すと、起動中の `./bin/sweep run → run_lbm_dem.py` のプロセス連鎖ごと終了します。

### 結果を確認する

- テーブルの行を**クリック**すると右下 **Results** パネルに:
  - **保存先パス**(「パスをコピー」「フォルダを開く」ボタン付き)
  - `summary.json` の主要指標(粒子数・最大速度・総ステップ数 など)
  - 時系列スパークライン(`particle_count` などの推移)

## 5. 複数マシンでの典型ワークフロー

各マシンで:

```bash
git pull                      # 最新の進捗を取り込む
poetry install --with web     # 初回のみ
./bin/sweep-web               # ブラウザで開く
```

1. まず誰かが **Status** を見て、どのケースが `pending` かを把握。
2. マシンごとに担当ケースを決めてチェック → 実行(`--no-push` を外す)。
3. 完了すると他マシンの画面にも `git pull` 後に `done` として反映される。

> ケース分担は手動です。同じケースを 2 台で同時実行しないよう、実行前に Status を確認してください。

---

## 6. 注意点・既知の制約

- **制御はローカル限定**: 実行/停止は Web アプリを起動した**そのマシン**でのみ機能します(進捗の*表示*だけが全マシン分)。各 PC で `./bin/sweep-web` を開いてください。
- **停止後のステータス**: 停止(SIGTERM)では CLI が `STARTED.json` を消さないため、再実行するまで他マシンからは `in-progress` のまま見えます。
- **resume は無い**: 中断したケースの再実行は step 0 からです(ソルバーにチェックポイント機構なし)。
- **進捗の粒度**: 進捗は `snapshot_every` ステップごと(例 1250 ステップ)にしか更新されません。それより細かくは出ません。
- **長時間ジョブ**: 本番ケースは数時間かかります。ブラウザを閉じても実行は継続します(プロセスはサーバ側で動作)。
- **ランタイム状態**: ログとローカル PID 台帳は gitignore 済みの `.sweep_web/` に保存されます。

## 7. トラブルシューティング

| 症状 | 対処 |
|---|---|
| 起動時に `ModuleNotFoundError: fastapi` | `poetry install --with web` を実行 |
| 実行を押すと `409 Conflict` | 同じケースが既に実行中。Status で確認 |
| 実行を押すと `400` でケース名エラー | マニフェストに無い alias/ケース名。`./bin/sweep list` で確認 |
| 結果が古いまま | 完了直後はハーベスト前。数秒待つか画面を更新 |
| ポート使用中 | `SWEEP_WEB_PORT=別番号 ./bin/sweep-web` |
